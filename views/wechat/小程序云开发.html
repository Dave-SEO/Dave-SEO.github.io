# 云开发优势

  • 快速上线
  • 专注核心业务
  • 独立开发一个完整的微信小程序
  • 不需要学习新的语言，只需要掌握JavaScript
  • 无需运维，节约成本
  • 数据安全
  • 无需域名与服务器（自带腾讯云服务）

# 云开发基础能力

  • 云函数（类似后端）：在云端运行的代码，微信私有协议天然鉴权
  • 云数据库：一个既可以在小程序端操作又可以在云函数中操作的JSON数据库
  • 云存储：在云端存储文件，可以在云端控制台可视化管理
  • 云调用：基于云函数免鉴权使用小程序开放接口的能力
  • HTTP API：使用HTTP API 开发者可在已有服务器上访问云资源，实现于云开发的互通

# tcb-router

  • 一个用户在一个云环境中只能创建50个云函数
  • 相似的请求归类到同一个云函数处理
  • tcb-router 是一个koa风格的云函数路由库

# 渲染层和逻辑层

  • 小程序的渲染层和逻辑层分别由2个线程管理：渲染层的界面使用了WebView 进行渲染；逻辑层采用JsCore线程运行JS脚本。一个小程序存在多个界面，所以渲染层存在多个WebView线程，这两个线程的通信会经由微信客户端做中转，逻辑层发送网络请求也经由Native转发
  • 所以频繁的setData 会造成资源消耗，如某些值不需要在页面使用，就不要在data属性中定义，外部定义变量即可
  • 避免setDate数据量过大，限制最大1M

# 小程序性能与优化

  • 合理设置可点击元素的区域大小
  • 避免渲染页面耗时过长
  • 避免执行脚本耗时过长
  • 网络请求做缓存
  • 不要引入未使用的wxss、wxml
  • 所以资源使用HTTPS
  • 避免过大的wxml节点数目
    1. 一个页面少于1000个wxml节点
    2. 节点数深度少于30层
    3. 子节点树不大于60个
  • 避免短时间发起太多的图片请求
  • 避免setData 的数据过大
  • 避免setData 频繁调用
  • 使用小程序开发工具中控制台提供的 audits 检测可以优化的点

# 小程序检测是否有新版本

    const updateManager = wx.getUpdateManager()
    updateManager.onCheckForUpdate(res => {
        if(res.hasUpdate){
            updateManager.onUpdateReady(() => {
                wx.showModal({
                    title: '提示',
                    content: '当前有新版本',
                    success(res){
                        if(res.confirm){
                            updateManager.applyUpdate()
                        }
                    }
                })
            })
        }
    })

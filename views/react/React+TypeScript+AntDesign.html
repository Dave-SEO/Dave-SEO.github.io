# react 的设计理念

  • 单项数据流
  • 虚拟dom 在js和dom之间增加一个ui抽象层，如果真实ui需要更新，react会对比两个ui层差距 以最小的差距，更新ui元素
  • 组件化
    1. 保持交互一致性
    2. 保持视觉风格统一
    3. 便于程序员相互协作

# 为什么使用JSX

  • React 认为视图的本质就是渲染逻辑与UI视图表现的内在统一
  • React 把HTML与渲染逻辑进行耦合，形成JSX

# JSX命名

  • 使用camelCase（小驼峰）方式定义属性
  • 自定义属性，以data-开头

# JSX 表示对象

  • jsx会被编译为React.createElement()对象

# JSX 中防止注入攻击

  • React DOM 在渲染内容时，会默认进行转义，变为字符串，可以有效防止XSS（跨站脚本）攻击
  • 花括号直接引用渲染内容时，React会自动转义

# css module（模组化）

  • 每个jsx或者tsx文件就被视为一个独立存在的原件
  • 原件所包含的所有内容也同样都应该是独立存在的

# TS 的定义声明

  • *.d.ts
  • 只包含类型声明，不包含逻辑
  • 不会被编译、也不会被webpack打包

# 在tsx 中使用css（css in js（jss））

  • 把css转化为对象，通过对象的方式，模块化引入css
  • import './xxx.css' => import style from './xxx.css'
  • 对css文件进行对象化声明 .d.ts文件
    // 创建custom.d.ts文件
    // 表示以import引入以css为后缀的文件时都会遵循以下的约定
    // 约定的内容是将会导出key所在的对象，而原始的类名和相应的值都会转化为这个对象
    declare module "*.css" {
        const css: {[key:string]:string};
        export default css;
    }

# State 和 Props区别

  • props 是组件对外的接口，而state是组件对内的接口
  • props 用于组件间的数据传递，而state用于组件内部的数据传递
  • State 的更新是异步的
    1. 调用setState 后，state不会立刻改变，是异步操作
    2. 不要依赖当前state，计算下个state

# target 与currentTarget

  • target 描述的是事件发生的元素
  • currentTarget 描述的是事件处理绑定的元素

# setState 是同步的还是异步的？

  • 异步更新，同步执行
  • setState() 本身并非异步，但对state的处理机制给人一种异步的假象。state处理一般发生在生命周期变化的时候
    this.setState({count: this.state.count + 1}, 
    () => {console.log(1)})
    
    this.setState((prevstate, prevprops) =>{return {count: this.state.count+1}} )

# React 组件的生命周期-类组件（初始化->更新->销毁）

# 初始化（循序依次执行）

  • 构建函数 初始化组件的state
  • getDerivedStateFromProps 探测state props 是否有变化
  • render() 渲染UI
  • componentDidMount: render执行完后就会调用componentDidMount，在组件创建好dom元素后，挂载进页面时调用

# 更新（循序依次执行）

  • getDerivedStateFromProps
  • shouldComponentUpdate(nextProps,nextState){}
  1. 更新阶段调用，通过nextProps,nextState变化控制组件是否更新
  2. 返回值为bol值
  3. 如需更新调用render：渲染UI
  • componentDidUpdate
    1. 组件更新后调用，只要组件更新，ui界面重新渲染就会执行

# 销毁（循序依次执行）

  • componentWillUnmount
  1. 组件销毁后调用

# React 17 （没有新功能，但允许多版本混用，为以后新版本平滑升级做准备）

  • 虽然没有新功能，但具有战略意义
  • 替换了底层的代码，向下兼容、同时甩掉了历史包袱
  • 没有代码断层，给未来留下了足够的升级空间

# 渐进式升级

  • 允许多个react版本共存
  • 先升级路由系统，再升级弹窗，平滑过度到最新版本

# Breaking Changes

  • 事件委托机制改变 在react16之前react会执行 dom.addEventListener,在document上添加事件处理器，而17版本不会向document上添加任何事件处理器，而会将事件处理器添加到渲染的react虚拟dom容器中，可更好的进行react新旧版本过渡
  • 向原生浏览器靠拢
  • 删除事件池
  • useEffect（本身是异步操作，但过去版本清理操作是同步的，这种不统一可能拖慢ui渲染效率） 清理操作改为异步操作
  • JSX 不可返回 undefined 在17以后必须有明确返回对象的
  • 删除部分私有API（大部分是供reactnative使用）

# 预测react 18

  • 更加强调函数式组件
  • 可能支持微前端架构

# 函数式组件

# 什么是Hooks（目的为不在写class component）

  • 消息处理的一种方法，用来监视指定程序
  • 函数组件中需要处理副作用，可以用钩子把外部代码钩进来
  • 常用钩子：useState，useEffect,useContext,useReducer
  • hooks 一律使用use前缀命名：usexxx
  • hooks本质是一类特殊的函数，为函数式组件注入特殊的功能

# 为什么要有Hooks

  • 有些类组件冗长而且复杂，难以复用
  • 尤其在与redux连接后无法复用，所以在没有hooks之前的解决方案为使用 无状态组件与HOC（高阶组件），但还是存在诸多问题
  • hooks的目的就是为了给函数式组件加上状态

# 自定义hooks

  • hook 是函数
  • 命名以use开头
  • 内部可调用其他hook函数
  • 并非react的特性

# 什么是纯函数

  • 给一个函数同样的参数，那么这个函数永远返回相同的值
  • 函数式编程的概念
  • React组件输入相同的参数，渲染UI应该永远一样
  • 好处容易检测数据，可控，渲染性能高

# 什么是副作用

  • 副作用与纯函数相反，指一个函数处理了与返回值无关的事情
  • 例如ajax，dom 操作， console.log
  • 例如:react: state状态的改变，生命周期，构建函数

# HOC是什么意思

  • 高阶组件（HOC）是React中用于复用组件逻辑的一种高级技巧。参数为组件，返回值 为新组件的函数

# 为什么要使用高阶组件（HOC）

  • 抽取重复代码，实现组件复用
  • 条件渲染，控制组件的渲染逻辑（渲染劫持）
  • 捕获/劫持被处理组件的生命周期

# 高阶组件使用以with（xxx）开头

# useState

  • React自带的一个hook函数，声明组件状态
  • 参数可以设置state的初始值（initial state）
  • 返回值是一个只有两个元素的数组：[状态，状态更新函数]
 const [count, setCount] = useState(0)
1

# useEffect

  • 可以取代生命周期函数 componentDidMount,componentDidUpdate 和 com potentWillUnmount
  • 给函数式组件添加副作用（side effect）
 useEffect(() => {document.title = `点击${count}次`}, [count])
1
  • useEffect 的第二个参数如果为[] 相当于模拟类组件的生命周期函数componentDidMount
  • useEffect 的第二个参数如果为空 相当于模拟类组件的生命周期函数componentDidUpdate

# Redux的使用

    创建文件 redux/store.ts
    import { createStore } from 'redux'
    import 
        {LanguageActionTypes, LANGUAGE_CHANGE }
     from './languageActions'
    import languageReducer from './language/languageReducer'
    const store = createStore(languageReducer)
    export default store
    // 创建文件redux/language/languageReducer.ts
    export interface LanguageState {
        language: 'en'| 'zh',
        languageList: {name:string, code: string}[]
    }

    const defaultState: LanguageState = {
        language: 'zh',
        languageList: [
            {name: '中文', code: 'zh'},
            {name: 'English', code: 'en'}
        ]
    }
    
    export default (state = defaultState, action: LanguageActionTypes) => {
    switch (action.type) {
        case LANGUAGE_CHANGE:
           return {...state, language: action.payload}
        default:
           return state
    }
}

// 创建文件redux/language/languageActions.ts
export const LANGUAGE_CHANGE = 'language_change'
export interface ChangeLanguageAction {
    type: typeof LANGUAGE_CHANGE;
    payload: 'zh' | 'en';
}
export const changeLanguageActionCreator = (languageCode: 'zh' | 'en') : ChangeLanguageAction => {
    return {
        type: LANGUAGE_CHANGE,
        payload: languageCode
    }
}

// 页面类组件中使用
import 
    { changeLanguageActionCreator } 
    from '../../redux/language/languageActions';
 import 
     { LanguageState } 
 from '../../redux/language/languageReducer';
 import store from '../../redux/store'; 
interface PropsType {}
interface state extends LanguageState {}
 class HeaderComponent extends React.Component<PropsType, state> {
  constructor(props){
        super(props)
        const storeState = store.getState()
        this.state = {
            language: storeState.language,
            languageList: storeState.languageList
        }
        //订阅subscribe 
        store.subscribe(this.subscribeHandler)
    }
    subscribeHandler = () => {
        const storeState = store.getState()
        this.setState({
            language: storeState.language
        })
    }
    menuClickHandler = (e) => {
        console.log('menuClickHandler', e)
        const action = changeLanguageActionCreator(e.key)
        store.dispatch(action)
    }
 }

// redux/hooks.ts 抽离 useSelector state 的类型定义组件更好复用
    import 
    {
        useSelector as useReduxSelector,
        TypedUseSelectorHook
    } from 'react-redux'
    import {RootState} from './store'
    export const useSelector: 
            TypedUseSelectorHook<RootState> = useReduxSelector;
    
// useSelector 从store中连接state数据
    import {useSelector} from '../../redux/hooks'
    // useDispatch 用于分发数据
    import {useDispatch} from 'react-redux'
    // 引入actionCreator
    import {changeLanguage} from '../../redux/actionCreator'
    // state 参数的类型定义在上面hooks.ts 所有state. 可以借助ts来获取提示
    const language = useSelector(state => state.language)
    const dispatch = useDispatch();
    
    // 事件中
    dispatch(changeLanguage('en'))

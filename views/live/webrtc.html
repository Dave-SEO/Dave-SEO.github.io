# webRTC 简介（https://appr.tc）

  1. 2010年google开源跨平台用于浏览器实时传输音视频引擎
  2. 音视频处理+即时通讯的开源库

# webRTC应用

  1. 照相机，音乐播放器，共享远程桌面，录制，即使通讯工具，p2p网络加速文件传输工具，游戏，实时人脸识别，音视频会议，在线教育

# webRTC设备管理

  1. enumerateDevices
  • 基本格式
var ePromise = navigator.mediaDevices.enumerateDevices();
if(!navigator.mediaDevices || !navigator.mediaDevices.enumerateDevices){
    console.log('设备不支持mediaDevices')
} else{ 
    var mediaDevice = navigator.mediaDevices.enumerateDevices()
    .then(getDevices)
    .catch(handleError)
}
function getDevices(deviceInfos){
    deviceInfos.forEach(deviceInfo => {
        console.log(`设备的种类-${deviceInfo.kind}设备的名称-${deviceInfo.label}-设备ID-${deviceInfo.deviceId}-groupId-${deviceInfo.groupId}`)
    });
}
function handleError(e){
    console.log(e.name + e.message)
}

# MediaStreamConstraints

 dictionary MediaStreamConstraints{
     (boolean or MediaTrackConstraints) video = false;
     (boolean or MediaTrackConstraints) audio = false;
 }

 'use strict'
if(!navigator.mediaDevices || !navigator.mediaDevices.getUserMedia){
    console.log('设备不支持getUserMedia')
}else{
    var constraints = {
        audio: true,
        video: true
    }
    var mediaStream = navigator.mediaDevices.getUserMedia(constraints)
    .then(gotMediaStream)
    .catch(mediaStreamError)
}
function mediaStreamError(e){
    console.log(e.message)
}
function gotMediaStream(stream){
    var video = document.getElementById('video');
    video.srcObject = stream
}

  • 使用Google开源库 adapter.js (https://webrtc.github.io/adapter/adapter-latest.js)
  1. webRTC音视频采集约束
    1. width,height,
    2. aspectRatio (描述用于一个视频轨迹的宽高比的可接受的或所需的值（一个或多个）。该值是宽度除以高度，并四舍五入到十位小数。 例如，16：9的标准高清视频宽高比可以计算为1920/1080或1.7777777778)
    3. frameRate(帧率，可以控制码流，如果帧率低会看到画面不平滑，帧率大码流大，相当于每秒采集的数据变多) 可以设置min max
    4. facingMode（控制摄像头翻转）
      • user 前置摄像头
      • environment 后置摄像头
      • left 前置左侧摄像头
      • right 前置右侧摄像头
    5. resizeMode （采集的画面要不要裁剪）
    6. volume (0.0~1.0)
    7. sampleRate (音频采样率如：48000，36000，8000)
    8. sampleSize（音频采样率，一般16位，也就是两个字节）
    9. echoCancellation（回音消除，值为布尔值）
    10. autoGainControl（自定义增益，在原有声音的基础上是否增加音量）
    11. noiseSuppression（降噪，在采集数据时是否要开启降噪功能，值为布尔值）
    12. latency
    • 设置latency小，延迟小，当网络不好时会出现花屏，卡顿现象等问题，好处是可以实时通讯
    • 设置latency大，画面，声音更平滑，及时性不好
    1. channelCount（单声道双声道，一般为单声道，如果是乐器为双声道）
    2. deviceID (设备ID)
    3. groupID （设备组ID，若组ID相同则是同一套设备）

# webRTC录制

options   限制选项 -
1. mimeType   指定录制的是音频还是视频，录制的格式是什么
   - video/webm
   - audio/webm
   - video/webm;codecs = vp8

   - audio/webm;codecs = opus

# mediaRecorder API

  1. MediaRecorder.start(timeslice)
  • 没有选择timeslice，则它所有的录制数据都会保存到buffer里去
  • 如果设置timeslice，会按照时间切片存储数据
  1. MediaRecorder.stop()
  • 停止录制，此时会触发包括最终blob数据的dataavailable事件
  1. MediaRecorder.pause
  • 暂停录制
  1. MediaRecorder.resume()
  • 恢复录制
  1. MediaRecorder.isTypeSupported()
  • 检查可以录制的格式 --mp4,mp3...

# MediaRecorder 事件

  1. MediaRecorder.ondataavailable
  • 当数据有效时触发
  • 每次记录一定时间的数据时（如果没有指定时间片，则记录整个数据时）会定期触发
  1. MediaRecorder.onerror
  • 当有错误发生时，录制会被停止

# Socket.IO发送消息

  1. 给本次连接发消息 socket.emit()
  2. 给某个房间内所有人发送消息 io.in(room).emit() io 为所有的节点
  3. 除本连接外，给某个房间内所有人发送消息 socket.to(room).emit()
    • socket 为创建连接时的socket
  4. 除本连接外，给所有人发送消息 socket.broadcast.emit()

# socket.io客户端处理消息

  1. 发送action命令
    • S: socket.emit('action');
    • C: socket.on('action',function(){...});
  2. 发送一个action命令，带数据
    • S: socket.emit('action',data);
    • C: socket.on('action',function(data){...});
  3. 发送action命令，还有两个数据
    • S: soket.emit(action,arg1,arg2);
    • C: socket.on('action',function(arg1,arg2){...})
  4. 发送一个action命令，在emit方法中包含回调函数
    • S: socket.emit('action',data,function(arg1,arg2){...});
    • C: socket.on('action', function(data,fn){'a','b'})
